📌 Project Overview

This project is a full-stack AI-powered movie recommendation system. The user types a natural language description — like "mind-bending sci-fi with time travel" — and the app returns the 3 most semantically similar movies from the database.

Unlike keyword search, this system understands the meaning behind words using vector embeddings generated by OpenAI's text-embedding-3-small model.

⚙️ How It Works — Step by Step

Phase 1: Seeding the Database (POST /api/seed)

1

Connect to PostgreSQL on Neon

API reads the connection string from appsettings.json and opens a connection via Npgsql.

2

Enable pgvector & Create Table

Runs CREATE EXTENSION IF NOT EXISTS vector and creates the movies table with an embedding VECTOR(1536) column.

3

Generate Embeddings with OpenAI

For each of the 5 movies, sends a combined text (title + genre + description) to OpenAI's text-embedding-3-small model. Receives a 1536-float vector back.

4

Insert into PostgreSQL

Stores the title, genre, description, and the float vector in the database via Dapper's ExecuteAsync.

Phase 2: Recommending Movies (POST /api/recommend)

1

User types a query in React

e.g. "crime family drama" — React sends POST /api/recommend with {"query": "crime family drama"}.

2

Convert query to embedding vector

The API sends the query text to the same OpenAI model. Gets back a 1536-dimensional float vector representing the meaning of the query.

3

pgvector similarity search

Runs SQL using <-> (L2 distance, for ordering) and <=> (cosine distance, for score). Returns the 3 closest movies.

4

Return results to React

JSON array of 3 movies with title, genre, description, and similarity (0–1 score). React renders them as cards.

The Similarity SQL Query

SELECT title, genre, description,
       1 - (embedding <=> CAST(@Embedding AS vector)) AS similarity
FROM movies
ORDER BY embedding <-> CAST(@Embedding AS vector)
LIMIT 3
OperatorMeaningUsed For
<->L2 (Euclidean) distanceORDER BY — closest vectors first
<=>Cosine distanceCalculating similarity % (0–1)

🧩 Key Challenges Solved

Challenge 1 — Swagger Not Working on .NET 10

ProblemThe default Microsoft.AspNetCore.OpenApi package from the template doesn't provide a Swagger UI browser.

SolutionReplaced with Swashbuckle.AspNetCore (AddSwaggerGen(), UseSwagger(), UseSwaggerUI()) which provides the full interactive UI.

Challenge 2 — OpenAI SDK Breaking Change (v1 → v2)

ProblemOld code used EmbeddingsEndpoint.CreateEmbeddingAsync() — this API was removed in SDK v2.x.

SolutionUpdated to client.GetEmbeddingClient("model").GenerateEmbeddingAsync(text) — the new v2.x pattern.

Challenge 3 — pgvector Type Casting with Dapper

ProblemDapper passes a float[] as a plain array — PostgreSQL couldn't automatically cast it to the vector type, throwing a type mismatch error.

SolutionAdded explicit CAST(@Embedding AS vector) in the SQL, forcing the correct type conversion.

Challenge 4 — CORS Blocking the React Frontend

ProblemBrowser blocked API calls from localhost:5173 to localhost:5142 — different ports means CORS policy violation.

SolutionAdded a named CORS policy "AllowReact" with WithOrigins("http://localhost:5173") and applied it before routing with app.UseCors().

Challenge 5 — Secrets Committed to Git, GitHub Blocked Push

Problembin/Debug/net10.0/appsettings.json (containing the OpenAI API key) was committed before .gitignore was created. GitHub's secret scanning blocked every push attempt.

SolutionCreated a fresh orphan branch with git checkout --orphan — a single clean commit with no history, then force-pushed to replace main on GitHub entirely.

Challenge 6 — Raw String Literal SQL Indentation Error

ProblemC# raw string literals ("""...""") require the closing """ to set the indentation baseline. Misaligned whitespace caused a compiler error.

SolutionAligned all SQL lines to the same indentation level as the closing """ delimiter.
